    6. Create `src/features/reports/hooks/useReportGeneration.ts`:
       - State machine with states: `'checking' | 'generating' | 'viewing' | 'error'`
       - On mount with `simulationId` param: call `reportApi.check(simulationId)` via useQuery
       - When check returns `has_report=true`: set state to `'viewing'`, store the report_id (from `report_status` or fetch via `reportApi.getBySimulation`)
       - When check returns `has_report=false`: call `reportApi.generate({ simulation_id: simulationId })` via useMutation, on success set reportId + state to `'generating'`
       - While state=generating: poll `reportApi.getProgress(reportId)` via `usePollingQuery` at 2000ms interval with `isComplete: (data) => ['completed', 'failed', 'error'].includes(data.data.status)`
       - On progress complete: transition to `'viewing'`
       - On progress failed: transition to `'error'`
       - Fetch full report via `useQuery(['report', reportId], () => reportApi.get(reportId))` when state=viewing and reportId is set
       - Return: `{ state, reportId, progress, report, retry }` where retry re-triggers generation with `force_regenerate: true`
       - Handle edge case: if check fails (network error), set state to 'error'

    7. Create `src/features/reports/hooks/useScrollSpy.ts`:
       - `useScrollSpy(sectionIds: string[]): string | null`
       - Create IntersectionObserver with `rootMargin: '-80px 0px -60% 0px'` and `threshold: 0`
       - Observe all elements matching the provided sectionIds
       - Return the id of the first intersecting entry from the top
       - Set initial active section to `sectionIds[0]` if available
       - Re-create observer when `sectionIds` changes (useEffect dependency)
