# Phase 2 — UI Design Contract

> Visual and interaction contract for the app shell, i18n framework, stepper layout, and loading/error patterns. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (to be initialized during implementation) |
| Preset | default (Tailwind CSS v4 mode) |
| Component library | Radix UI (via shadcn/ui) |
| Icon library | Lucide React |
| Font | Geist (sans) via `@fontsource-variable/geist` |

shadcn must be initialized with `npx shadcn@latest init` before any component installation. Tailwind CSS v4 uses CSS-first configuration (no `tailwind.config.js`).

### shadcn Components Required for Phase 2

- `button` -- CTAs, retry buttons, quick-start
- `skeleton` -- loading placeholders
- `toast` (via `sonner`) -- global error/success notifications
- `badge` -- backend status indicator, step status
- `separator` -- visual dividers
- `tooltip` -- icon-only buttons (language switcher, config)
- `breadcrumb` -- header breadcrumb navigation
- `collapsible` -- sidebar sub-step expansion
- `sheet` -- mobile/tablet sidebar drawer
- `dropdown-menu` -- language switcher dropdown
- `alert` -- inline error/warning messages

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, icon-to-label gap |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, sidebar item height |
| xl | 32px | Layout column gaps, content area padding |
| 2xl | 48px | Major section breaks, header height |
| 3xl | 64px | Page-level vertical spacing, sidebar width anchor |

Exceptions:
- Sidebar width: 280px (desktop expanded), 64px (collapsed icon-only)
- Header height: 56px
- Footer height: 40px
- Content max-width: 960px for centered form views, full-width for simulation/graph views
- Touch target minimum: 44px for interactive elements on tablet

## Typography

Font: Geist Sans (`--font-geist-sans`). Load weights 400 and 600 only.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 15px | 400 (regular) | 1.5 | Paragraph text, descriptions, empty state body |
| Label | 13px | 600 (semibold) | 1.4 | Form labels, sidebar step names, badges, breadcrumbs, footer |
| Heading | 22px | 600 (semibold) | 1.2 | Page titles, section headings, dashboard card titles |
| Display | 30px | 600 (semibold) | 1.1 | Dashboard hero heading only |

Polish strings are 20-30% longer than English. All text containers must accommodate this expansion without truncation or layout shift. Test with Polish strings from day one.

## Color

Light mode only in Phase 2. Dark mode deferred to Phase 9.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` / white | Page background, content area |
| Secondary (30%) | `hsl(240 5% 96%)` | Sidebar background, card surfaces, skeleton fill, footer background |
| Accent (10%) | `hsl(217 91% 60%)` / blue-500 | See reserved list below |
| Destructive | `hsl(0 84% 60%)` / red-500 | Destructive actions only (none in Phase 2) |

### Accent Reserved For

- Active/current stepper step indicator (left border or filled dot)
- Primary CTA buttons ("Rozpocznij" / quick-start)
- Language switcher active state
- Backend status "connected" dot (green variant: `hsl(142 71% 45%)`)
- Breadcrumb current page text
- Links in body text

### Semantic Colors

| State | Value | Usage |
|-------|-------|-------|
| Success | `hsl(142 71% 45%)` / green-500 | Backend connected dot, completed step indicator |
| Warning | `hsl(38 92% 50%)` / amber-500 | Not used in Phase 2, reserved |
| Error | `hsl(0 84% 60%)` / red-500 | Backend disconnected dot, error toasts, inline error alerts |
| Muted text | `hsl(240 4% 46%)` | Placeholder text, disabled labels, footer text |

### Border

- Default border: `hsl(240 6% 90%)` / 1px solid
- Border radius: 8px for cards and containers, 6px for buttons and inputs, 12px for modals/sheets

## Layout Contract

### Desktop (>= 1280px)

```
+--[Header 56px]---------------------------------------------+
| [Logo]  [Breadcrumb: Dashboard > ...]  [Lang] [Config] [●] |
+----------+--------------------------------------------------+
| Sidebar  | Content Area                                      |
| 280px    | padding: 32px                                     |
| fixed    | max-width: varies by stage                        |
|          |                                                   |
| [Steps]  |                                                   |
|          |                                                   |
+----------+--------------------------------------------------+
| [Footer 40px: v0.0.1 | GitHub | AGPL-3.0]                  |
+-------------------------------------------------------------+
```

### Tablet (1024px - 1279px)

- Sidebar collapses to icon-only rail (64px) by default
- Sidebar expands to full 280px overlay (Sheet component) on hamburger click
- Content area takes remaining width
- All spacing tokens remain the same

### Below 1024px

- Not a target. Display a message: "Ta aplikacja wymaga ekranu o szerokosci co najmniej 1024px" / "This application requires a screen width of at least 1024px"

### Sidebar Stepper

- Each stage is a row: icon + label + status badge (completed/current/locked)
- Current stage: accent-colored left border (3px), semibold label
- Completed stage: success-colored check icon, regular weight label, clickable
- Locked stage: muted text, lock icon, not clickable
- Sub-steps expand on click via Collapsible component
- Sub-step items indented 24px from parent

### Header

- Height: 56px, white background, bottom border 1px
- Left: App logo/name (Geist 15px semibold)
- Center: Breadcrumb showing navigation path
- Right group (8px gap): Language dropdown, Settings icon-button with tooltip, Backend status dot (8px circle)

### Dashboard View

- Centered content, max-width 640px
- Hero section: Display heading + Body subheading
- System status card: backend connection status, Docker status placeholder
- Recent simulations list: empty state when none
- Quick-start CTA button: accent background, white text, full-width on mobile

### Footer

- Height: 40px, secondary background
- Content: version number (left), GitHub link (center), AGPL-3.0 note (right)
- Label typography (13px), muted text color

## Loading States

| Context | Pattern | Details |
|---------|---------|---------|
| Page/route transition | Skeleton | 3 skeleton blocks mimicking content layout, pulse animation at 1.5s cycle |
| Dashboard cards | Skeleton | Card-shaped skeleton with rounded corners matching card radius |
| Action button click | Spinner | 16px spinner replaces button label, button disabled during loading |
| Backend status check | Dot blink | Status dot blinks at 1s interval while checking |
| Sidebar stages | Skeleton | 5 skeleton rows matching step height (44px each) |

Skeleton fill color: `hsl(240 5% 90%)` with pulse to `hsl(240 5% 96%)`.

---

## Error States

| Context | Pattern | Details |
|---------|---------|---------|
| Network error (global) | Toast | Sonner toast, bottom-right position, auto-dismiss after 5s, includes retry action |
| Backend unreachable | Toast + status dot | Red status dot in header + persistent toast (no auto-dismiss) with retry |
| Auto-retry exhausted | Toast upgrade | After 2 auto-retries, toast becomes persistent with manual "Sprobuj ponownie" button |
| Chinese backend error | Mapped toast | Error intercepted by `mapChineseError()` from `src/api/errors.ts`, displayed in current language |
| Route not found | Full page | Centered message with "Wróć do strony głównej" / "Return to home" link |

Toast configuration: Sonner provider, position bottom-right, max 3 visible, stack direction up. Rich colors enabled for semantic toast types (success green, error red).

## Animation Contract

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| Page/route transition | Fade in | 150ms | ease-out |
| Sidebar expand/collapse | Width slide | 200ms | ease-in-out |
| Sheet (mobile sidebar) | Slide from left | 200ms | ease-out (shadcn default) |
| Toast enter | Slide up + fade | 200ms | ease-out (Sonner default) |
| Toast exit | Fade out | 150ms | ease-in (Sonner default) |
| Skeleton pulse | Opacity oscillation | 1500ms | ease-in-out, infinite |
| Stepper sub-step expand | Height reveal | 200ms | ease-out |
| Backend status dot blink | Opacity 1 to 0.3 | 1000ms | ease-in-out, infinite (only while checking) |

No motion preference: respect `prefers-reduced-motion: reduce` -- disable all animations except opacity fades.

## i18n Contract

| Property | Value |
|----------|-------|
| Framework | react-i18next + i18next |
| Default language | pl (Polish) |
| Available languages | pl, en |
| Plural forms | Polish: 4 forms via ICU MessageFormat (one, few, many, other) |
| Namespace structure | JSON per namespace per language: `locales/{lang}/{namespace}.json` |
| Namespaces for Phase 2 | `common`, `navigation`, `dashboard`, `errors` |
| Fallback | en (English) |
| Detection | Browser language, then localStorage override |
| Interpolation | Double curly braces `{{variable}}` (i18next default) |

### Translation File Structure

```
src/
  locales/
    pl/
      common.json      # Shared: buttons, labels, status
      navigation.json   # Sidebar steps, breadcrumbs, header
      dashboard.json    # Dashboard page copy
      errors.json       # Error messages, Chinese error mappings
    en/
      common.json
      navigation.json
      dashboard.json
      errors.json
```
