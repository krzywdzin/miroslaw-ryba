    **1. Create `src/features/graph/lib/graph-transforms.ts`:**
    - `toReagraphNodes(apiNodes: Array<{ id: string; name: string; type?: string }>): GraphNode[]`
      Maps each node to Reagraph format: `{ id, label: name, fill: ENTITY_COLORS[type ?? 'default'], data: { type, name } }`
      Import `ENTITY_COLORS` from `./graph-colors`
    - `toReagraphEdges(apiEdges: Array<{ source: string; target: string; relation?: string }>): GraphEdge[]`
      Maps each edge: `{ id: \`${source}-${target}-${i}\`, source, target, label: relation }`
    - Import `GraphNode, GraphEdge` types from `reagraph`

    **2. Create `src/features/graph/hooks/useGraphData.ts`:**
    - TanStack Query hook wrapping `graphApi.getGraphData(graphId)`
    - Query key: `['graph', 'data', graphId]`
    - `enabled: !!graphId`
    - `staleTime: Infinity` (graph data does not change after construction)
    - Returns the query result with convenience: `{ nodes, edges, isLoading, error }`
    - Apply `toReagraphNodes` and `toReagraphEdges` transforms in a `select` option so downstream components receive Reagraph-formatted data directly

    **3. Create `src/features/graph/hooks/useGraphFilters.ts`:**
    - Local state hook (NOT Zustand -- filter state is ephemeral):
      - `activeTypes: Set<string>` (initialized with all ENTITY_TYPES enabled)
      - `searchQuery: string` (default: '')
      - `toggleType(type: string): void`
      - `setSearchQuery(q: string): void`
    - `getFilteredData(nodes: GraphNode[], edges: GraphEdge[])` function:
      - Filter nodes where `data.type` is in activeTypes
      - If searchQuery non-empty, filter further to nodes where label contains searchQuery (case-insensitive)
      - Filter edges to only those whose source AND target are in the filtered node set
      - Return `{ filteredNodes, filteredEdges, highlightedNodeIds }` where highlightedNodeIds are nodes matching search (for visual emphasis)

    **4. Create `tests/features/graph/graph-transforms.test.ts`:**
    Tests:
    - toReagraphNodes maps name to label
    - toReagraphNodes applies correct fill color for 'person' type
    - toReagraphNodes uses default color when type is undefined
    - toReagraphNodes stores type and name in data field
    - toReagraphEdges generates unique id from source-target-index
    - toReagraphEdges maps relation to label
    - toReagraphEdges handles undefined relation (label undefined)
    - Filter logic: activeTypes filter removes nodes of excluded type
    - Filter logic: search query filters by node label case-insensitively
    - Filter logic: edges are pruned when their source or target is filtered out
  </action>
