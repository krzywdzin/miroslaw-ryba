### Dark Mode Implementation

- Strategy: Tailwind v4 `@custom-variant dark (&:where(.dark, .dark *))` in globals.css
- Class toggle: add/remove `dark` class on `<html>` element
- FOUC prevention: inline `<script>` in index.html `<head>` reads localStorage and applies class before React mounts
- Print CSS: existing `body { background: white !important; color: black !important; }` in `@media print` overrides dark mode for PDF export

### Accent Reserved For (both modes)

- Active stepper step indicator (left border)
- Primary CTA buttons
- Language switcher active state
- Theme switcher active state (new)
- Breadcrumb current page text
- Links in body text
- Focus ring on interactive elements

## Theme Switcher Contract

### Component: ThemeSwitcher

| Property | Value |
|----------|-------|
| Location | Header, right group, next to LanguageSwitcher |
| Pattern | DropdownMenu trigger with icon Button (ghost variant, size icon) |
| States | 3: System / Jasny (light) / Ciemny (dark) |
| Default | System (follows OS `prefers-color-scheme`) |
| Persistence | localStorage key `theme` |
| Icons | Sun (light active), Moon (dark active), Monitor (system option) |
| Active indicator | `bg-accent text-accent-foreground` on selected menu item |

### Theme State Flow

1. On app load: inline script reads `localStorage.getItem('theme')`, applies `dark` class if needed
2. React mounts: `useTheme` hook reads localStorage, syncs state
3. User clicks toggle: `setTheme()` updates localStorage + toggles class on `<html>`
4. OS preference changes: if theme is "system", re-evaluate and update class
5. System option: `localStorage.removeItem('theme')` (absence = system)

### Theme Transition

| Property | Value |
|----------|-------|
| Transition | None (instant switch) |
| Reason | CSS custom properties update is inherently instant; adding transition-all causes visible paint delay on complex pages |

### Shortcut Map

| Key | Action | Scope |
|-----|--------|-------|
| `1` | Navigate to /graph (Budowa grafu) | Global, disabled in inputs |
| `2` | Navigate to /environment (Srodowisko) | Global, disabled in inputs |

| `5` | Navigate to /chat (Dialog) | Global, disabled in inputs |
| `/` | Focus chat input (`[data-chat-input]`) | Global, disabled in inputs |

| `]` | Switch to next panel/tab on current page | Global, disabled in inputs |
| `?` | Open shortcuts help modal | Global, disabled in inputs |

### Implementation

| Property | Value |
|----------|-------|
| Library | react-hotkeys-hook v5.2.4 |
| Hook | `useHotkeys` with `enableOnFormTags: false` |
| Registration | `useGlobalShortcuts` hook called in App.tsx |
| Input guard | Automatic via `enableOnFormTags: false` -- shortcuts disabled when focus is in `<input>`, `<textarea>`, `<select>`, or `[contenteditable]` |

| Title (PL) | Skroty klawiszowe |
| Title (EN) | Keyboard Shortcuts |

## FOUC Prevention Contract

| Property | Value |
|----------|-------|

### Dark Mode Visual Audit

Every page must be verified in both light and dark mode. Check for:
- Hardcoded color classes (`bg-white`, `text-black`, `bg-gray-*`, `border-gray-*`)

- Chart/graph colors readable on dark backgrounds
