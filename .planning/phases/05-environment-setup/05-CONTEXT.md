# Phase 5: Environment Setup - Context

**Gathered:** 2026-03-21
**Status:** Ready for planning

<domain>
## Phase Boundary

Users can review extracted entities, agent profiles, and simulation parameters before launching simulation, completing pipeline stage 2. This is a review/confirmation step — users verify what the system understood and adjust parameters before committing to simulation.

</domain>

<decisions>
## Implementation Decisions

### Entities and Relationships
- Sortable, filterable table for entities: columns for name, type, relationship count
- Clicking an entity shows its relationships as a list in a detail view (not mini-graph)
- Filter by entity type, sort by name/type/relationship count

### Agent Profiles
- Rich cards showing: avatar (Lucide icon by agent type), name, personality traits, memory state, stance
- Grid layout: 3-4 columns on desktop
- Cards are read-only — agents are generated by backend, not user-editable

### Simulation Parameters
- Per-simulation override: config has defaults, this page allows per-run adjustments
- Summary card showing current values ("Agenci: 20, Rundy: 10, Platformy: Twitter+Reddit")
- "Dostosuj" button opens inline form with sliders/switches (same pattern as /settings)
- Changes here don't persist to global config — they're per-simulation only

### Page Layout
- Tab-based: three tabs — Encje | Agenci | Parametry
- Not scrollable single page — each tab focuses on one concern

### Transition to Simulation
- "Uruchom symulację" CTA button at the bottom of the page (visible on all tabs)
- Clicking opens confirmation dialog with parameter summary before starting
- After confirmation, auto-transition to simulation monitoring page (Phase 6)

### Claude's Discretion
- Entity table column widths and responsive behavior
- Agent card exact layout and info density
- Confirmation dialog content and layout
- Tab indicator style
- How entity detail view is presented (side panel, modal, expandable row)
- Loading states while environment data loads from backend

</decisions>

## Specific Ideas

- Tab-based layout keeps each concern focused — entity table can be wide without competing with cards
- Summary card + "Dostosuj" pattern avoids overwhelming users who just want to review defaults
- Confirmation dialog before simulation gives a last chance to review — prevents accidental expensive operations
- Rich agent cards with type icons make the simulated world tangible and inspectable
