| Property | Value |

## Interaction Patterns

### Auto-Save Behavior

- Trigger: form field change (debounced 500ms after last keystroke)
- Validation: on blur (mode: 'onBlur' in React Hook Form)
- Persistence: Zustand store with persist middleware writes to localStorage under key `mirofish-config`
- Feedback: subtle success toast ("Zapisano") lasting 1500ms, bottom-right position
- Invalid fields: do NOT save, show inline validation error, no toast

### Connection Test Flow

1. User clicks "Testuj" / "Testuj polaczenie"
2. Button shows spinner, is disabled
3. Request sent (LLM: chat completion with `max_tokens: 5`; Zep: REST API session query)
4. On success: green checkmark icon + success message appears below input
5. On failure: red X icon + specific error message appears below input
6. Result persists until user clicks test again or changes the input value

### Docker Compose Confirmation

1. "Zatrzymaj backend" shows as ghost/outline button
2. On click: button text changes to "Na pewno?" with destructive color for 3 seconds
3. If clicked again within 3 seconds: action executes
4. If not clicked within 3 seconds: reverts to original "Zatrzymaj backend" text
5. During execution: spinner on button, all Docker controls disabled

### Scroll Spy for Section Navigation

- IntersectionObserver watches each section heading (`id` attribute matches anchor)
- When a section heading enters viewport top 30%: corresponding nav item becomes active
- Active state: accent text color + 2px accent left border
- Clicking a nav item: smooth-scrolls to the section, updates URL hash
