       - Create a Vite plugin that intercepts requests to `/docker-api/*`
       - Forwards to Docker Unix socket at `/var/run/docker.sock`
       - Uses Node.js `http` module to make requests to the Unix socket
       - Strips `/docker-api` prefix before forwarding (e.g., `/docker-api/containers/json` -> `/containers/json`)
       - For compose commands (up/down), exec `docker compose -f ./mirofish/docker-compose.yml up -d` or `down` via Node.js `child_process.execSync`
       - Add special routes: POST `/docker-api/compose/up` and POST `/docker-api/compose/down`
       - Handle errors gracefully: if Docker socket not found, return 503 with JSON error

    2. Create `src/features/settings/types/docker.types.ts`:
       - DockerContainer: { Id: string, Names: string[], Image: string, State: 'running' | 'exited' | 'paused' | 'restarting' | 'dead', Status: string }
       - DockerStats: { cpu_stats: { cpu_usage: { total_usage: number }, system_cpu_usage: number }, precpu_stats: { cpu_usage: { total_usage: number }, system_cpu_usage: number }, memory_stats: { usage: number, limit: number } }
       - ContainerAction: 'start' | 'stop' | 'restart'

    3. Create `src/features/settings/hooks/useDockerStatus.ts`:
       - `useDockerStatus()`: uses usePollingQuery to poll GET `/docker-api/containers/json?all=true` every 10 seconds
         - isComplete always returns false (continuous polling)
         - Returns { containers, isLoading, isError, error }
         - Stop polling when component unmounts (handled by usePollingQuery)
       - `useContainerStats(containerId: string, enabled: boolean)`: polls GET `/docker-api/containers/${id}/stats?stream=false` every 10 seconds
         - Returns parsed CPU percentage and memory usage/limit
         - CPU calculation: (cpu_delta / system_delta) * 100
       - `useContainerLogs(containerId: string, enabled: boolean)`: fetches GET `/docker-api/containers/${id}/logs?stdout=true&stderr=true&tail=100` as text
         - Returns { logs: string, refetch }
         - Refetches every 10 seconds when enabled
       - `useContainerAction()`: useMutation for POST `/docker-api/containers/${id}/${action}`
         - Invalidates container list query on success
         - Shows error toast via handleApiError on failure
       - `useComposeAction()`: useMutation for POST `/docker-api/compose/${action}` (up/down)
         - Invalidates container list query on success
